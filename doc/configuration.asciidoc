== Overview

The configuration system should support three levels of configuration (in order of precedence):

. Command-line flags (highest priority)
. Environment variables
. Configuration file (`~/.config/qws/config.yaml`)

== Naming Conventions

The configuration system uses different naming conventions for each format, with automatic conversion handled by Viper.

=== Rules

[cols="1,2,3"]
|===
| Format | Convention | Example

| **YAML**
| `snake_case` with dots for nesting
| `keybindings.workspace_modifier`

| **CLI Flags**
| All dashes (dots and underscores → dashes)
| `--keybindings-workspace-modifier`

| **Environment Variables**
| `UPPER_SNAKE_CASE` with `QWS_` prefix (dots → underscores)
| `QWS_KEYBINDINGS_WORKSPACE_MODIFIER`
|===

=== Conversion Examples

.Complete conversion chain
[cols="3,3,4"]
|===
| YAML Key | CLI Flag | Environment Variable

| `keybindings.modifier`
| `--keybindings-modifier`
| `QWS_KEYBINDINGS_MODIFIER`

| `keybindings.workspace_modifier`
| `--keybindings-workspace-modifier`
| `QWS_KEYBINDINGS_WORKSPACE_MODIFIER`

| `appearance.colors.dark.selection_frame`
| `--appearance-colors-dark-selection-frame`
| `QWS_APPEARANCE_COLORS_DARK_SELECTION_FRAME`

| `behavior.snapshot_interval`
| `--behavior-snapshot-interval`
| `QWS_BEHAVIOR_SNAPSHOT_INTERVAL`

| `windows.ignore_skip_taskbar`
| `--windows-ignore-skip-taskbar`
| `QWS_WINDOWS_IGNORE_SKIP_TASKBAR`
|===

=== Implementation Details

The conversion is handled automatically by Viper:

[source,go]
----
// Setup Viper for environment variables
viper.SetEnvPrefix("QWS")
viper.AutomaticEnv()
viper.SetEnvKeyReplacer(strings.NewReplacer(".", "_"))

// Bind CLI flags to config keys
viper.BindPFlag("keybindings.workspace_modifier",
               cmd.Flags().Lookup("keybindings-workspace-modifier"))
----

**Key points:**

* YAML keys with underscores (`workspace_modifier`) are preserved as-is internally
* Dots (`.`) separate nesting levels in YAML and are converted to underscores for ENV
* CLI flags convert both dots and underscores to dashes for GNU/POSIX compliance
* Viper handles all conversions transparently — just use the YAML key in code

== Configuration Categories

=== Keybindings

.Current hardcoded values in `pkg/keygrab/keygrab.go`
[cols="2,2,3"]
|===
| Parameter | Default | Description

| `modifier`
| `Alt` (Mod1)
| Main modifier key

| `key`
| `Tab`
| Main trigger key

| `backward_modifier`
| `Shift`
| Modifier for reverse navigation

| `workspace_modifier`
| `Ctrl`
| When NOT pressed, shows windows from all workspaces; when pressed together with main modifier, filters to current workspace only

| `cancel_key`
| `Escape`
| Key to cancel selection and close switcher
|===

.Proposed CLI flags
[source,bash]
----
qws --modifier=Super --key=grave
qws -m Alt -k Tab
qws --workspace-modifier=Super  # Use Super+Alt+Tab to filter current workspace
----

.Proposed config
[source,yaml]
----
keybindings:
  modifier: Alt        # Alt, Super, Ctrl, Mod1-Mod5
  key: Tab             # Tab, grave, space, etc.
  backward: Shift
  workspace_modifier: Ctrl  # Filter to current workspace when held
  cancel: Escape
----

=== Window Filtering

.Current behavior in `pkg/x11/windows.go`
- Excludes `_NET_WM_WINDOW_TYPE_DOCK`
- Excludes `_NET_WM_WINDOW_TYPE_DESKTOP`
- Excludes windows with `_NET_WM_STATE_SKIP_TASKBAR`

.Proposed config
[source,yaml]
----
windows:
  desktop: current           # current, all, all-except-current
  ignore_skip_taskbar: false # Respect _NET_WM_STATE_SKIP_TASKBAR hint

  #   - "polybar"

=== Usage Examples

==== YAML Configuration

[source,yaml]
----
# ~/.config/qws/config.yaml
keybindings:
  modifier: Super
  key: grave

appearance:
  thumbnail:
    width: 192
    height: 192
  colors:
    theme: dark
    dark:
      selection_frame: "#ff6b6b"

behavior:
  snapshot_interval: 5s

windows:
  desktop: all
----

==== Command-Line Interface

[source,bash]
----
# Use default configuration
qws

# Use custom config file
qws --config ~/.config/qws/work.yaml
qws -c ~/.config/qws/work.yaml

# Override keybindings
qws --keybindings-modifier=Super --keybindings-key=grave
qws -m Super -k grave  # short form

# Change appearance
qws --appearance-thumbnail-width=192 --appearance-thumbnail-height=192

# Force dark theme with custom selection color
qws -t dark --appearance-colors-dark-selection-frame="#ff6b6b"

# Configure behavior
qws --behavior-snapshot-interval=5s --windows-desktop=all

# Debug mode
qws -v   # debug level
qws -vv  # trace level
----

==== Environment Variables

[source,bash]
----
# Set keybindings
export QWS_KEYBINDINGS_MODIFIER=Super
export QWS_KEYBINDINGS_KEY=grave

# Set appearance
export QWS_APPEARANCE_THUMBNAIL_WIDTH=192
export QWS_APPEARANCE_COLORS_THEME=dark
export QWS_APPEARANCE_COLORS_DARK_SELECTION_FRAME="#ff6b6b"

# Set behavior
export QWS_BEHAVIOR_SNAPSHOT_INTERVAL=5s

# Set logging
export QWS_LOG_LEVEL=debug

# Run with environment variables
qws
----

==== Combined Configuration

[source,bash]
----
# Config file sets base configuration
# Environment variables override specific settings
# CLI flags have highest priority

# Base: ~/.config/qws/config.yaml (theme: auto, modifier: Alt)
# Override theme via env:
QWS_APPEARANCE_COLORS_THEME=dark \
# Override modifier via CLI (highest priority):
qws --keybindings-modifier=Super
----

== Theme System

QWS will support automatic theme switching based on system preferences.

=== Theme Detection

The application will detect the system theme using the following methods (in order):

. **Environment variable**: `QWS_APPEARANCE_COLORS_THEME` or `GTK_THEME`
. **GTK settings**: `~/.config/gtk-3.0/settings.ini` → `gtk-application-prefer-dark-theme`
. **XSETTINGS**: X11 `Net/ThemeName` property via XSETTINGS daemon
. **Fallback**: light theme

=== Custom Colors

Users can override individual colors for each theme while keeping automatic theme switching:

[source,yaml]
----
appearance:
  colors:
    theme: auto  # Auto-switch between themes
    dark:
      selection_frame: "#ff6b6b"  # Custom selection color for dark theme
    light:
      selection_frame: "#4a9eff"  # Custom selection color for light theme
----

Partial overrides are supported — only specified colors will be customized, others will use defaults:

[source,yaml]
----
appearance:
  colors:
    theme: auto
    dark:
      selection_frame: "#ff6b6b"  # Only override selection frame
      # background, text, shadow, inactive_frame use defaults
----

== Design Decisions

[cols="1,2"]
|===
| Decision | Rationale

| No X resources support
| Viper provides sufficient configuration layers (CLI, env, file). X resources is a legacy mechanism that doesn't work on Wayland and adds unnecessary complexity.

| No profile support
| Single config file only — either default (`~/.config/qws/config.yaml`) or specified via `--config` flag. Keeps implementation simple.

| No explicit daemon mode
| Application already runs as a background service. Use `systemctl --user` for proper daemon management (autostart, restart policies, logging).

| Auto theme switching
| Follow system theme by default (`theme: auto`). Users can override via config/CLI/env. Theme detection uses GTK settings as primary source for X11 compatibility.
|===
